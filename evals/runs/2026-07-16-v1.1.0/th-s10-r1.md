## Direction

**วิทยานิพนธ์การออกแบบ:** พอร์ตโฟลิโอสำหรับผู้ว่าจ้างและผู้ร่วมงานสายสร้างสรรค์ ที่พาคนดูทำความรู้จักตัวตนและตัดสินคุณภาพงานผ่านลำดับเรื่องแบบ editorial อสมมาตร โทนอุ่นเป็นมนุษย์ และภาพผลงานพร้อมหลักฐานจากกระบวนการจริง

ใช้กริดแบบเงียบเป็นโครงเบื้องหลัง ไม่แสดงเส้นกริดหรือ technical chrome เป็นบุคลิกหลัก ภาพงานต้องได้พื้นที่และสเกลเพียงพอ ไม่ถูกลดเป็น thumbnail ในการ์ดซ้ำ ๆ รักษางานศิลป์ ข้อความบรรยาย หลักฐานของแต่ละโปรเจกต์ และป้ายเปิดเผยว่างานใดเป็นงานสมมติไว้ตามต้นฉบับ ห้ามแต่งลูกค้า ตัวเลขผลลัพธ์ คำชม หรือหลักฐานใหม่เพื่อให้ผลงานดูน่าเชื่อขึ้น

## ระบบภาพและเนื้อหา

- **องค์ประกอบ:** ใช้คอนเทนเนอร์ที่มีขอบนอกสม่ำเสมอและ 6–12 คอลัมน์ตามขนาดจอ แต่ให้ภาพสลับกินพื้นที่ประมาณ 5/7, 7/5 หรือเต็มความกว้าง สร้างความอสมมาตรอย่างมีจังหวะ ไม่ทำ masonry แบบสุ่ม
- **ตัวอักษร:** ใช้ display face ที่มีอารมณ์อบอุ่นหรือรายละเอียดเชิงหัตถกรรมคู่กับ body face ที่อ่านง่าย จำกัดบทบาทเป็น display, heading, body, caption และ metadata ไม่ใช้ monospace เว้นแต่เนื้อหาเป็นโค้ดหรือข้อมูลเทคนิคจริง
- **สีและพื้นผิว:** เริ่มจากสีแบรนด์เดิม ถ้าไม่มี ให้ใช้พื้นสีงาช้างหรือกระดาษ หมึกเข้ม และสีเน้นที่มาจากงานศิลป์ ใช้ texture อย่างเบามากเพื่อเพิ่มสัมผัส ไม่ให้รบกวนตัวอักษรหรือคอนทราสต์
- **รูปทรง:** หลีกเลี่ยงกรอบการ์ดและมุมโค้งแบบเดียวทั่วหน้า ใช้เส้นแบ่งเฉพาะเมื่อช่วยอธิบายลำดับหรือความสัมพันธ์ และปล่อยให้พื้นที่ว่างกับ crop ของภาพสร้างบุคลิก
- **ภาพ:** กำหนด crop ต่อชิ้นอย่างตั้งใจ แต่เปิดทางให้ดูภาพเต็มพร้อมคำบรรยาย ระบุ alt text ที่บอกสาระของงาน; ภาพตกแต่งให้ซ่อนจากเทคโนโลยีช่วยเหลือ
- **การเคลื่อนไหว:** ใช้เฉพาะเพื่อเผยลำดับหรือเชื่อมภาพกับรายละเอียด เช่น crossfade สั้นหรือการขยับเล็กน้อยเมื่อเปิดโปรเจกต์ รองรับ `prefers-reduced-motion` และไม่ใช้ cursor แปลก ๆ, scroll hijacking หรือ canvas ที่ทำให้คีย์บอร์ดและการซูมใช้งานไม่ได้

## โครงหน้า

1. **เปิดเรื่อง:** ภาพเด่นหนึ่งชิ้นครองจอ วางชื่อ บทบาท และประโยคแนะนำสั้น ๆ เยื้องจากภาพอย่างตั้งใจ มีลิงก์ตรงไปยังผลงานและข้อมูลติดต่อ ไม่ใช้ hero copy แบบกึ่งกลางทั่วไป
2. **Selected work:** เล่าเป็นลำดับโปรเจกต์ที่มีองค์ประกอบต่างกัน—บางชิ้นใช้ภาพเต็ม บางชิ้นใช้ภาพคู่กับข้อความ บางชิ้นใช้รายละเอียดกระบวนการ—แต่ยึดแนวเส้นและระยะชุดเดียวกัน ทุกชิ้นแสดงชื่อ บทบาท ปี บริบท และสถานะ **“งานสมมติ / Concept project”** ใกล้ชื่อเมื่อเกี่ยวข้อง
3. **Project narrative:** เปิดด้วยภาพหลัก ตามด้วยโจทย์ บทบาท ข้อจำกัด การตัดสินใจ กระบวนการ และผลลัพธ์ที่พิสูจน์ได้ วางสเก็ตช์ ภาพระหว่างทำ ภาพหน้าจอ หรือ artifact ต้นฉบับข้างข้อความที่มันสนับสนุน ไม่แยกหลักฐานไปเป็นแกลเลอรีไร้บริบท
4. **About:** ใช้ภาพบุคคลหรือภาพโต๊ะทำงานจริงร่วมกับข้อความเสียงเป็นธรรมชาติ กล่าวถึงวิธีทำงานและสิ่งที่สนใจอย่างเฉพาะเจาะจง ไม่ใช้ชุด skill pills หรือค่าความเชี่ยวชาญที่ไม่มีหลักฐาน
5. **Contact:** ปิดด้วยคำเชิญที่ชัดเจนและเป็นมนุษย์ พร้อมช่องทางติดต่อจริง ลิงก์ทุกอันมีชื่อที่สื่อความหมายและสถานะโฟกัสมองเห็นได้

## Responsive และการเข้าถึง

บนจอใหญ่ให้ความอสมมาตรมาจากสเกล ภาพเหลื่อม และจังหวะพื้นที่ว่าง บนแท็บเล็ตลดจำนวนแนวจัดวางและรักษาลำดับเรื่อง บนมือถือเรียงเนื้อหาตามความสำคัญ—ชื่อและสถานะงาน, ภาพ, บทสรุป, หลักฐาน—ไม่ย่อองค์ประกอบ desktop ทั้งชุดลงมา ภาพต้องไม่ล้นโดยไม่ตั้งใจ คำบรรยายยังอยู่ติดกับภาพ และพื้นที่กดไม่น้อยเกินไป

ใช้ semantic landmarks, ลำดับ heading ที่ถูกต้อง, ลิงก์สำหรับข้ามไปเนื้อหา, คีย์บอร์ดโฟกัสที่เห็นชัด, คอนทราสต์เพียงพอ และสถานะที่ไม่พึ่งสีอย่างเดียว หากมี lightbox ต้องปิดด้วย Escape กักโฟกัสขณะเปิด และคืนโฟกัสเมื่อปิด

## เกณฑ์รับงาน

- ผู้ชมระบุได้อย่างรวดเร็วว่าเจ้าของพอร์ตคือใคร ทำงานอะไร และควรเริ่มดูตรงไหน
- โปรเจกต์แต่ละชิ้นมีรูปแบบการเล่าที่เหมาะกับเนื้อหา ไม่ถูกบังคับเป็นการ์ดแม่แบบเดียวกัน
- งานศิลป์ คำบรรยาย และหลักฐานเดิมยังครบและเชื่อมโยงกัน
- งานสมมติทุกชิ้นมีป้ายเปิดเผยชัดเจนในรายการและหน้ารายละเอียด
- หน้าเว็บยังใช้งานด้วยคีย์บอร์ด การซูม และ reduced motion ได้ โดยไม่พึ่ง experimental canvas
- ไม่เพิ่มคำกล่าวอ้าง ลูกค้า เมตริก หรือผลลัพธ์ที่ไม่มีหลักฐาน

นี่เป็นข้อกำหนดทิศทางการออกแบบจาก brief เท่านั้น ยังไม่มีหน้าที่ render หรือ interaction จริงให้สังเกต จึงยังไม่อ้างว่า implementation, responsive behavior หรือ accessibility ได้รับการตรวจสอบแล้ว
