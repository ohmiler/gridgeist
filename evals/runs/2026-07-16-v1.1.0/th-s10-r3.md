## ทิศทาง

**Thesis:** พอร์ตโฟลิโอสำหรับนักว่าจ้างและผู้ร่วมงานสายสร้างสรรค์ ใช้ลำดับแบบนิตยสารที่ไม่สมมาตร แบรนด์อบอุ่นและเป็นมนุษย์ โดยให้งานศิลป์ขนาดใหญ่ คำบรรยายจากเจ้าของงาน และหลักฐานของกระบวนการเป็นตัวพาผู้อ่านไปรู้จักทั้งฝีมือและวิธีคิด

สิ่งที่ต้องรักษาไว้โดยไม่ต่อรองคือ งานศิลป์ต้นฉบับ ครอปที่มีความหมาย น้ำเสียงของคำบรรยาย หลักฐานของโปรเจกต์ และคำเปิดเผยว่างานใดเป็นงานสมมติ การออกแบบใหม่ไม่ควรทำให้สิ่งเหล่านี้กลายเป็น thumbnail เล็ก ๆ หรือข้อความประกอบการ์ดที่เหมือนกันทุกชิ้น

กริดควรเป็น **invisible/quiet grid**: ใช้กำหนดขอบหน้า แนวข้อความ และจังหวะภาพอย่างสม่ำเสมอ แต่ไม่แสดงเส้นกรอบ โมโนสเปซ ป้ายสถานะ หรือ technical chrome เป็นบุคลิกหลัก ความอสมมาตรเกิดจากสเกล ตำแหน่ง การเหลื่อม และพื้นที่ว่างที่ตั้งใจ ไม่ใช่จากการวางองค์ประกอบแบบสุ่ม

## ระบบและองค์ประกอบ

- หน้าเปิดใช้ภาพงานเด่นหนึ่งชิ้นเป็นพระเอก กินพื้นที่ราวสองในสามของ viewport คู่กับข้อความแนะนำสั้น ๆ ที่เขียนด้วยเสียงของนักพัฒนาเอง CTA มีเพียง “ดูผลงาน” และ “ติดต่อ” โดยไม่ใช้ hero แบบข้อความกึ่งกลาง
- ลำดับโปรเจกต์เป็นบทบรรณาธิการสลับจังหวะ ไม่ใช่กริดการ์ดเท่ากัน: งานหนึ่งอาจเปิดด้วยภาพเต็มขอบ อีกงานเปิดด้วยคำถามของโจทย์ หรือด้วย artefact จากกระบวนการ แต่ทุกบทใช้โครงข้อมูลร่วมกันเพื่อไม่ให้หลงทาง
- แต่ละโปรเจกต์ควรตอบให้ครบว่า “โจทย์คืออะไร / ฉันรับผิดชอบอะไร / ทำอย่างไร / มีหลักฐานอะไร / ผลลัพธ์หรือสิ่งที่เรียนรู้คืออะไร” ใช้ภาพหน้าจอ สเก็ตช์ prototype บันทึกการตัดสินใจ หรือโค้ดจริงเมื่อมี แทนคำกล่าวอ้างกว้าง ๆ ห้ามสร้างลูกค้า ตัวเลขผลลัพธ์ หรือคำรับรองขึ้นมาเอง
- งานสมมติต้องมีข้อความชัดเจนใกล้ชื่อและบริบท เช่น **“โปรเจกต์แนวคิด — ไม่ได้ว่าจ้างและไม่ได้ใช้งานจริง”** ป้ายนี้ต้องอ่านได้ด้วยข้อความ ไม่พึ่งสีอย่างเดียว และไม่ถูกซ่อนไว้ท้ายหน้า
- ใช้ display type ที่มีลักษณะอ่อนโยนหรือมีรายละเอียดเชิงมนุษย์ร่วมกับ body type ที่อ่านสบาย จำกัดบทบาทตัวอักษรให้ชัดเจน และเก็บโมโนสเปซไว้เฉพาะโค้ดหรือข้อมูลเทคนิคที่เป็นของจริง
- สีเริ่มจากพาเลตเดิมของแบรนด์: พื้นโทนอุ่น สีหมึกที่ไม่ดำแข็ง และสีเน้นจากงานศิลป์ รูปร่าง ขอบ เงา และ texture มีเท่าที่ช่วยการแบ่งชั้นหรือสัมผัสของแบรนด์ ไม่ผสมกรอบหนัก เงาหนัก gradient และมุมโค้งพร้อมกันโดยไร้เหตุผล
- motion ควรตอบสนองการเปิดเผยภาพ การเปลี่ยนลำดับ หรือ feedback ของลิงก์อย่างสั้นและสงบ ไม่มี ambient motion ที่แย่งงานศิลป์ และต้องมีทางเลือกเมื่อเปิด `prefers-reduced-motion`

## โครงหน้า

1. **Opening portrait** — ภาพงานเด่น ชื่อ/บทบาท ประโยคแนะนำที่เฉพาะตัว และทางไปผลงาน
2. **Selected work** — 3–5 โปรเจกต์ที่มีองค์ประกอบต่างจังหวะกัน แต่ใช้ anatomy ของเรื่องเล่าร่วมกัน
3. **Process interlude** — ภาพโต๊ะทำงาน สเก็ตช์ commit excerpt หรือบันทึกสั้น ๆ ที่เผยวิธีคิดโดยไม่ทำเป็น dashboard
4. **About** — ภาพบุคคลหรือภาพแวดล้อมจริง ชีวประวัติที่กระชับ ความสนใจ และรูปแบบความร่วมมือ
5. **Contact** — คำเชิญที่เป็นมนุษย์ ช่องทางตรง และสถานะการรับงานเมื่อมีข้อมูลจริง

## Responsive และการเข้าถึง

บนจอกว้างให้ภาพกับข้อความเหลื่อมกันอย่างควบคุมผ่านแนวร่วมไม่กี่แนว บน tablet ลดการเหลื่อมและรักษาลำดับเรื่อง ส่วน mobile ต้อง **recompose** เป็นภาพนำ → ชื่อ/สถานะ → บทสรุป → หลักฐาน ไม่ย่อเลย์เอาต์ desktop หรือปล่อยพื้นที่ว่างเชิงตกแต่งไว้ ภาพต้องมีพฤติกรรมครอปที่กำหนดต่อ breakpoint; งานที่ห้ามครอปควรใช้ `object-fit: contain` และมีพื้นหลังรองรับ

ใช้ semantic landmarks และลำดับ heading ที่ต่อเนื่อง ลิงก์และปุ่มต้องมี focus ที่มองเห็นได้ พื้นที่สัมผัสเพียงพอ รูปงานมี alt text ที่อธิบายสาระ ส่วน texture ตกแต่งถูกซ่อนจาก assistive technology คำบรรยายภาพต้องยังเชื่อมกับภาพเมื่อเลย์เอาต์เปลี่ยน และทุกเนื้อหาต้องใช้งานได้ด้วยคีย์บอร์ด

## สถานะการตรวจสอบ

นี่เป็นข้อเสนอเชิงออกแบบจากข้อกำหนดที่ให้มาเท่านั้น รอบนี้ไม่มี target fixture และไม่มีหน้าที่ render จึงยังไม่มีหลักฐานจาก viewport, interaction, keyboard, contrast, reduced motion หรือเนื้อหา dynamic และไม่ควรเรียกผลลัพธ์ว่า implement แล้วหรือผ่านการตรวจสอบ ขั้นถัดไปเมื่อมีตัวงานคือดูอย่างน้อยที่ประมาณ 360, 768, 1280 และ 1600 พิกเซล พร้อมตรวจ default, hover, focus, loading/error ของสื่อ และการอ่านคำเปิดเผยงานสมมติในทุก breakpoint
