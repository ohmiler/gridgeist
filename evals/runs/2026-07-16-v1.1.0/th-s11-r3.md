# Direction

ชื่อสมมติ: **ปุ๊บปั๊บวาด** — “สตูดิโอวาดรูปแบบสัมผัสสำหรับเด็ก 6–10 ปี จัดทุกอย่างรอบผืนวาด ใช้กลุ่มเครื่องมือที่เป็นมิตร และตอบสนองทันทีบนเครื่อง”

ผืนวาดเป็นพระเอกและกินพื้นที่มากที่สุด กริดทำหน้าที่เงียบ ๆ แบ่งเพียง 3 เขต: เครื่องมือสร้างรอย, ผืนวาด, และการจัดการภาพ ภาษาภาพอบอุ่นและขี้เล่นแต่ไม่ดูเป็นของเด็กเล็กเกินไป: พื้นหลังสีครีม, ตัวอักษร sans ทรงมนที่อ่านง่าย, เส้นขอบคล้ายกระดาษตัด และเงาสั้นเฉพาะสิ่งที่กดได้ ลวดลายประจำผลิตภัณฑ์คือ “รอยพู่กัน” ซึ่งใช้บอกสี ขนาด และสถานะจริง ไม่ใช่ของตกแต่ง

ข้อจำกัดที่รักษาไว้ตั้งแต่ระดับผลิตภัณฑ์: ไม่มีบัญชี, โปรไฟล์, ชื่อผู้ใช้, อัปโหลด, กล้อง, analytics, โฆษณา หรือการเก็บข้อมูลส่วนบุคคล งานวาดอยู่ในหน่วยความจำของแท็บระหว่างใช้งาน และออกจากแอปได้เฉพาะไฟล์ที่เด็กสั่งบันทึกลงเครื่องเอง หน้าแอปบอกสั้น ๆ ว่า “รูปอยู่ในแท็บนี้ จนกว่าจะกดบันทึก” โดยไม่ใช้ถ้อยคำรับรองทางกฎหมาย

# Implementation specification

## โครงหน้าจอ

- แถบบนสูงประมาณ 64 px: โลโก้คำ, Undo, Redo ทางซ้าย; “ล้างภาพ” และปุ่มหลัก “บันทึกรูป” ทางขวา ไอคอนทุกตัวมีคำกำกับเสมอ
- แถบเครื่องมือด้านซ้ายกว้างประมาณ 88 px บนจอใหญ่: “สี” อยู่เหนือ “ขนาด” และแสดงค่าที่เลือกอยู่ทันที
- ผืนวาดสีขาวอยู่กลางหน้าจอ มีขอบชัดและพื้นที่ว่างรอบขอบสำหรับวางฝ่ามือโดยไม่โดนเครื่องมือ เป้าหมายสัมผัสทุกจุดอย่างน้อย 48×48 px และปุ่มหลัก 56 px
- ไม่มี card ย่อยซ้ำ ๆ หรือแถบเมนูซ้อนหลายชั้น เด็กควรเริ่มวาดได้ทันทีโดยไม่ต้องผ่าน onboarding

## การเลือกสี

- แสดงสีหลัก 10 สีเป็นวงกลมขนาด 48 px ในกริด 2 คอลัมน์ สีที่เลือกมีทั้งวงแหวนหนา เครื่องหมายถูก และชื่อสีใน label จึงไม่พึ่งสีอย่างเดียว
- ด้านบนกลุ่มสีมี “รอยป้ายสี” ขนาดใหญ่ซึ่งแสดงสีจริงของพู่กันปัจจุบัน
- ปุ่ม “สีเพิ่ม” เปิด bottom sheet เป็นกริดสีเพิ่มเติม ไม่ใช้ color wheel ที่ต้องเล็งละเอียด และไม่มี eyedropper, การเปิดไฟล์ หรือกล้อง
- เมื่อแตะสี พู่กันเปลี่ยนทันทีและเกิด feedback แบบย่อ/เด้งเบา ๆ; หากลดการเคลื่อนไหว ระบบเปลี่ยนเป็นเส้นขอบหนาขึ้นโดยไม่มี animation

## ขนาดพู่กัน

- มี 4 ขนาดที่มองเห็นเป็นวงจริง: เล็ก, กลาง, ใหญ่, ใหญ่มาก แต่ละปุ่มมีทั้งจุดตัวอย่างและคำกำกับ
- ค่าเริ่มต้นคือ “กลาง” ซึ่งเหมาะกับการเริ่มลากเส้นโดยไม่ต้องตั้งค่า
- ขนาดที่เลือกแสดงในรอยป้ายสีด้านบนด้วย เด็กจึงเห็นผลของ “สี + ขนาด” ก่อนวาด
- ไม่ใช้ slider เป็นตัวควบคุมหลัก เพราะต้องเล็งและอ่านค่าละเอียดเกินจำเป็น

## วาดและกู้คืน

- แตะหรือลากหนึ่งนิ้วเพื่อวาด เส้นปรากฏต่อเนื่องใต้ปลายนิ้วทันที; รองรับเมาส์และ stylus โดยไม่เปลี่ยนโมเดลการใช้งาน
- Undo และ Redo เป็นปุ่มคงที่ ไม่ซ่อนใน gesture: Undo ย้อนหนึ่ง stroke และ Redo คืนหนึ่ง stroke กดค้างเพื่อทำซ้ำเป็นจังหวะได้ พร้อมประกาศสถานะ เช่น “ย้อนเส้นแล้ว”
- เมื่อไม่มีสิ่งให้ย้อนหรือทำซ้ำ ปุ่มยังอยู่ตำแหน่งเดิมแต่เป็น disabled พร้อมความต่างทั้งความทึบและรูปทรง ไม่ทำให้ layout กระโดด
- การวาด stroke ใหม่หลัง Undo จะล้างประวัติ Redo ตามพฤติกรรมมาตรฐาน และมีข้อความสั้น “เริ่มทางใหม่แล้ว”
- ป้องกันการ scroll/zoom เฉพาะภายในผืนวาด แต่ยังให้หน้าและเครื่องมือรอบนอกเลื่อนได้ตามปกติบนจอเล็ก

## ล้างภาพ

- “ล้างภาพ” ใช้สีเตือนและไอคอนไม้กวาด ไม่ใช้ถังขยะที่อาจสื่อว่าลบไฟล์
- หากผืนวาดว่าง ปุ่มเป็น disabled; หากมีภาพ แตะแล้วเปิด bottom sheet ว่า “ล้างรูปทั้งภาพไหม?” พร้อมปุ่ม “กลับไปวาด” เป็นตัวเลือกแรกและ “ล้างภาพ” เป็นตัวเลือกยืนยัน
- หลังล้าง แสดงแถบข้อความ “ล้างแล้ว — ย้อนกลับ” ชั่วคราว และ Undo ยังกู้ภาพทั้งหมดได้ จึงมีทั้งการยืนยันและทางกู้คืน

## Export ลงเครื่อง

- ปุ่ม “บันทึกรูป” ส่งออกผืนวาดเป็น PNG บนเครื่องด้วยการแปลง canvas ฝั่ง client แล้วเรียก download ของ browser; ไม่มี network request และไม่ส่งภาพไป server
- ชื่อไฟล์เป็นค่ากลาง เช่น `รูปวาด-2026-07-16.png` ไม่ถามชื่อเด็ก ชื่อโรงเรียน หรือ metadata ส่วนบุคคล และไม่ฝังข้อมูลตำแหน่ง
- ระหว่างสร้างไฟล์ ปุ่มเปลี่ยนเป็น “กำลังเตรียม…” และ disabled เพื่อกันกดซ้ำ เมื่อสำเร็จแสดง “บันทึกแล้ว” พร้อมไอคอนถูก เมื่อ browser ปฏิเสธการดาวน์โหลด แสดง “ยังบันทึกไม่ได้ ลองกดอีกครั้ง” โดยภาพบนผืนวาดไม่หาย
- หากผืนวาดว่าง ปุ่มบันทึกเป็น disabled พร้อมข้อความ “วาดอะไรสักนิดก่อน”

## Responsive

- ตั้งแต่ประมาณ 768 px ขึ้นไป: ผืนวาดอยู่กลาง แถบสี/ขนาดอยู่ซ้าย และการจัดการภาพอยู่บนสุด เหมาะกับแท็บเล็ตแนวนอนและคอมพิวเตอร์
- ที่ประมาณ 360–767 px: ผืนวาดขึ้นก่อนและเกือบเต็มความกว้าง; Undo/Redo เป็นแถบเล็กด้านบน ส่วนสีและขนาดย้ายเป็น dock ด้านล่างที่นิ้วโป้งเอื้อมถึง สีแสดงเป็นแถวเลื่อนแนวนอน และ “สีเพิ่ม” เปิด bottom sheet
- จอแนวตั้งไม่ย่อแถบซ้ายจนปุ่มเล็ก แต่จัดลำดับใหม่ เด็กยังเห็นผืนวาด สีปัจจุบัน ขนาด Undo และบันทึกโดยไม่ต้องเปิดเมนูซ่อน
- รองรับ keyboard ด้วย Tab order ตามงานจริง, focus ring ชัด, `Ctrl/Cmd+Z`, `Ctrl/Cmd+Shift+Z` และปุ่ม Enter/Space โดยไม่แทนที่การควบคุมแบบสัมผัส

## สถานะสำคัญ

- Default: ผืนวาดว่าง สีดำ ขนาดกลาง; Undo, Redo, ล้าง และบันทึกเป็น disabled
- Active/selected: ปุ่มยุบเล็กน้อยขณะกด และค่าที่เลือกมีวงแหวน + เครื่องหมาย ไม่อาศัยสีเพียงอย่างเดียว
- Success/error: ข้อความอยู่ใกล้การกระทำ ไม่บังผืนวาด และไม่ลบงานเมื่อ export ผิดพลาด
- Destructive: ล้างภาพต้องยืนยันและ Undo ได้
- Exit: หากมีงานที่ยังไม่ได้ export การปิดหรือรีโหลดใช้คำเตือนมาตรฐานของ browser; ไม่แอบบันทึกลง localStorage หรือ server

# Verification status

นี่เป็นข้อเสนอการออกแบบเท่านั้น ยังไม่ได้ implement, render หรือทดสอบ flow จริง จึงยังไม่มีหลักฐานเรื่องความง่ายในการใช้ การเข้าถึง ความปลอดภัย หรือผลการใช้งานกับเด็ก ก่อนพัฒนาจริงควรตรวจที่ความกว้างประมาณ 360, 768, 1280 และ 1600 px; ทดลอง touch, stylus, mouse และ keyboard; ตรวจ stroke latency, palm interaction, Undo/Redo branching, ล้างแล้วกู้คืน, export สำเร็จ/ถูกปฏิเสธ, focus, contrast, reduced motion และ overflow

ควรทำ usability testing กับเด็กและผู้ปกครอง/ผู้ดูแลตามกระบวนการที่เหมาะสมก่อนสรุปว่าใช้ง่ายสำหรับวัย 6–10 ปี และไม่กล่าวอ้างว่าเป็นไปตาม COPPA หรือข้อกำกับใด ๆ จนกว่าจะมีการประเมินโดยผู้เชี่ยวชาญและหลักฐานที่เกี่ยวข้อง
