# Direction

**Design thesis:** สตูดิโอวาดรูปแบบ touch-first สำหรับเด็ก 6–10 ปี จัดทุกอย่างรอบผืนผ้าใบ ใช้กลุ่มเครื่องมือที่เป็นมิตร ตำแหน่งคงที่ และ feedback ในเครื่องทันที โดยให้ “รอยพู่กัน” เป็น motif หลัก

ชื่อสมมติ **ปาดปุ๊บ (PaatPoop)** ใช้น้ำเสียงสั้น ตรง และให้กำลังใจโดยไม่ทำให้เด็กดูเล็กเกินไป ผืนผ้าใบกินพื้นที่มากที่สุดเสมอ ส่วน UI ใช้โครงสร้างเงียบ ไม่ตีเส้นกริดให้ดูเหมือนเครื่องมือเทคนิค สีของ UI เป็นกลางและอุ่น เพื่อไม่แย่งกับสีในภาพ; สีรุ้งใช้กับเครื่องมือเลือกสีเท่านั้น ตัวอักษรทรงมน อ่านง่าย ไม่ใช้ฟอนต์ลายมือกับข้อความควบคุม

ข้อกำหนดที่รักษาไว้เป็นพฤติกรรมของผลิตภัณฑ์: ใช้ได้ทันทีโดยไม่มีบัญชี ทุกอย่างทำงานในเครื่อง ไม่มีอัปโหลด กล้อง analytics โฆษณา social sharing หรือการเก็บข้อมูลส่วนบุคคล ปุ่ม export บันทึกไฟล์ลงอุปกรณ์ผ่านความสามารถมาตรฐานของเบราว์เซอร์เท่านั้น ไม่กล่าวอ้างว่าเป็นไปตาม COPPA และไม่กล่าวอ้างว่าออกแบบจากการทดสอบกับเด็กจนกว่าจะมีหลักฐานจริง

# โครงหน้าจอ

## จอแนวนอน/แท็บเล็ต

- แถบบนสูงประมาณ 64–72 px: โลโก้เล็กด้านซ้าย, ชื่องานแบบไม่ต้องกรอกค่าเริ่มต้นว่า “รูปของฉัน”, Undo และ Redo ตรงกลาง, “บันทึกรูป” ด้านขวา
- รางเครื่องมือซ้ายกว้าง 88–104 px: **สี**, **พู่กัน**, และ **ยางลบ** แต่ละปุ่มมีไอคอนพร้อมคำ ไม่พึ่งไอคอนล้วน
- ผืนผ้าใบเป็นพื้นที่หลักตรงกลาง มีขอบบางและพื้นหลังสีขาว ไม่วางข้อความหรือของตกแต่งทับพื้นที่วาด
- “ล้างภาพ” อยู่มุมล่างซ้าย แยกจากเครื่องมือที่ใช้บ่อย ใช้สีเตือนและไอคอนถังขยะพร้อมคำ เพื่อป้องกันการแตะพลาด

เป้าสัมผัสทุกจุดอย่างน้อย 48×48 px มีช่องว่างระหว่างปุ่มอย่างน้อย 8 px ไม่มี gesture ลับหรือการกดค้างที่จำเป็นต่อการทำงานหลัก และปิด browser pinch/scroll เฉพาะในขอบเขตผืนผ้าใบระหว่างกำลังวาด ไม่ปิดทั้งหน้า

## โทรศัพท์แนวตั้ง

ไม่ย่อ desktop ลงมาตรง ๆ: แถบบนเหลือ Undo, Redo และเมนู “เพิ่มเติม”; ผืนผ้าใบอยู่บนและสูงเท่าที่ viewport อนุญาต ส่วน dock ด้านล่างมี **สี**, **ขนาด**, **ยางลบ**, **บันทึก** แบบ 4 ช่อง ปุ่มล้างอยู่ในแผง “เพิ่มเติม” และต้องยืนยัน พาเลตสีหรือขนาดพู่กันเปิดเป็น bottom sheet สูงไม่เกินครึ่งจอ จึงยังเห็นรอยที่กำลังแก้อยู่

# ปฏิสัมพันธ์หลัก

## เลือกสี

แตะ **สี** แล้วเปิดพาเลต 12 สีขนาดใหญ่เป็นตาราง 4×3: ดำ น้ำตาล แดง ส้ม เหลือง เขียวอ่อน เขียวเข้ม ฟ้า น้ำเงิน ม่วง ชมพู และขาว/ยางลบ สีล่าสุด 4 สีอยู่แถวบนในจอกว้าง สีที่เลือกมีทั้งวงแหวนหนา เครื่องหมายถูก และข้อความชื่อสีสำหรับ screen reader จึงไม่ใช้สีเป็นสัญญาณอย่างเดียว

มีปุ่ม “สีเพิ่ม” เปิดวงล้อสีแบบง่ายพร้อมแถบความสว่าง แต่ไม่บังคับเด็กใช้ตัวเลือกซับซ้อน เมื่อเลือกแล้ว ปุ่มสีหลักเปลี่ยนเป็นหยดสีปัจจุบันและแสดงตัวอย่างเส้นสั้น ๆ

## ขนาดพู่กัน

แตะ **พู่กัน/ขนาด** แล้วเห็นตัวเลือกเพียง 5 ระดับ: จุดเล็ก, เส้นเล็ก, กลาง, ใหญ่, ใหญ่มาก แสดงเป็น “เส้นจริง” ไม่ใช่ตัวเลขพิกเซล แต่ละตัวเลือกมีชื่อและพื้นที่แตะเต็มแถว ระหว่างเลื่อนหรือแตะ ให้ preview เส้นสีปัจจุบันในช่องตัวอย่างที่ไม่เขียนลงผืนผ้าใบ ค่าเริ่มต้นคือ “กลาง”

## การวาด

วาดได้ทันทีด้วยนิ้ว ปากกา หรือเมาส์ เส้นตาม pointer อย่างต่อเนื่องและมีจุดเริ่มต้นแม้เพียงแตะหนึ่งครั้ง รองรับหลาย pointer โดยรับเส้นแรกและเพิกเฉยต่อการแตะที่สอง เพื่อไม่ให้ฝ่ามือสร้างรอยโดยไม่ตั้งใจ เคอร์เซอร์บนเดสก์ท็อปแสดงวงกลมตามขนาดพู่กัน; บนจอสัมผัสไม่เพิ่มเคอร์เซอร์ลอยที่บังปลายนิ้ว

เมื่อเลือกยางลบ ปุ่มและเคอร์เซอร์เปลี่ยนสถานะชัดเจน พร้อมข้อความ “กำลังลบ”; แตะพู่กันหรือสีใด ๆ เพื่อกลับไปวาด ไม่ใช้เสียงหรือ vibration เป็น feedback ที่จำเป็น และไม่มี motion ตกแต่งเหนือผืนผ้าใบ

## Undo / Redo

Undo และ Redo เป็นปุ่มแยกพร้อมลูกศรและคำบนจอกว้าง ทุก stroke หนึ่งเส้นคือหนึ่งขั้น Undo รวมทั้งการล้างภาพ Redo พร้อมใช้หลัง Undo และถูกปิดเมื่อเริ่มวาดเส้นใหม่ ปุ่มที่ใช้ไม่ได้เป็นสถานะ disabled จริง ลด contrast แต่ยังอ่านชื่อได้ ไม่หายไปจนตำแหน่งเปลี่ยน

หลังแตะ Undo/Redo แสดง toast สั้น ๆ เช่น “ย้อนเส้นล่าสุดแล้ว” โดยไม่บังภาพและไม่ขโมย focus รองรับคีย์ลัด Ctrl/Cmd+Z และ Ctrl/Cmd+Shift+Z แต่คีย์ลัดเป็นทางเสริม ไม่ใช่วิธีเดียว

## ล้างภาพ

แตะ **ล้างภาพ** แล้วเปิด dialog ตรงไปตรงมา: “ล้างทั้งรูปไหม?” พร้อมภาพย่อของงานปัจจุบัน ปุ่มหลัก “ยังไม่ล้าง” และปุ่มอันตราย “ล้างรูป” การกด Escape หรือแตะฉากหลังยกเลิกและคืน focus ไปยังปุ่มเดิม

เมื่อล้างแล้ว แสดงแถบด้านล่าง “ล้างรูปแล้ว — ย้อนกลับ” เป็นเวลา 8 วินาที และการกด Undo ภายหลังก็คืนภาพได้เช่นกัน จึงมีทั้งการยืนยันและ recovery ที่เป็นรูปธรรม ถ้าผ้าใบว่าง ปุ่มล้างเป็น disabled และไม่เปิด dialog

## Export ลงเครื่อง

ปุ่มใช้คำว่า **บันทึกรูป** แทนศัพท์ “Export” เมื่อแตะ เปิดแผ่นสรุปที่มีภาพตัวอย่างและตัวเลือกเพียงสองแบบ:

- **PNG — พื้นหลังเหมือนที่เห็น** เป็นค่าแนะนำ
- **PNG — พื้นหลังโปร่งใส** เป็นตัวเลือกเพิ่มเติม

ชื่อไฟล์เริ่มต้นเป็น `รูปของฉัน-YYYY-MM-DD.png` และแก้ได้โดยไม่ต้องกรอกชื่อบุคคล ปุ่มสุดท้าย “บันทึกลงเครื่อง” เรียกการดาวน์โหลดในเครื่องเท่านั้น เมื่อเบราว์เซอร์เริ่มดาวน์โหลด แสดง “ส่งรูปไปที่โฟลเดอร์ดาวน์โหลดแล้ว” พร้อมคำแนะนำ “หาไม่เจอ? เปิดรายการดาวน์โหลดของเบราว์เซอร์” โดยไม่กล่าวว่าบันทึกสำเร็จถ้า API แจ้งข้อผิดพลาด

สถานะ error ใช้ข้อความ “ยังบันทึกไม่ได้ ภาพของคุณยังอยู่ ลองอีกครั้ง” พร้อมปุ่ม “ลองใหม่” ไม่มีการเสนออัปโหลดหรือส่งให้ผู้ปกครองเป็น fallback

# สถานะที่ต้องออกแบบ

- **เริ่มต้น/ผ้าใบว่าง:** มีเพียงคำใบ้ชั่วคราวนอกผ้าใบ “เลือกสี แล้วลากนิ้วเพื่อวาด” ซึ่งหายหลัง stroke แรก
- **กำลังวาด:** controls คงตำแหน่ง ไม่มี panel โผล่หรือ layout shift
- **เลือกแล้ว:** สี ขนาด และยางลบมีสัญญาณทั้งรูปทรง/ข้อความ/สี
- **Disabled:** Undo, Redo และล้างภาพบอกเหตุผลด้วย accessible description
- **Destructive:** ล้างต้องยืนยันและย้อนกลับได้
- **Export กำลังเตรียม:** ปุ่มเปลี่ยนเป็น “กำลังเตรียมรูป…” และป้องกันการแตะซ้ำ แต่ไม่ล็อกการวาดนานเกินจำเป็น
- **Export สำเร็จ/ผิดพลาด:** ข้อความไม่คลุมเครือและมี recovery
- **หน่วยความจำไม่พอหรือ canvas มีปัญหา:** หยุดรับ stroke ใหม่ แจ้งว่า “พื้นที่วาดเต็มแล้ว ลองบันทึกรูปก่อน” โดยรักษาภาพเดิมเท่าที่ทำได้
- **ออกจากหน้าโดยยังมีงาน:** ใช้ browser leave warning เฉพาะเมื่อมี stroke ที่ยังไม่เคยบันทึก; หลังบันทึกหรือผ้าใบว่างไม่เตือน

# Accessibility และความเป็นส่วนตัว

ใช้ `<canvas>` ควบคู่กับชื่อและคำอธิบายที่เข้าถึงได้ พร้อม toolbar และ button semantics ที่ถูกต้อง ลำดับ focus เดินจากเครื่องมือหลักไปผืนผ้าใบและคำสั่งไฟล์ มี focus ring ชัดเจน ทุก dialog trap focus และคืน focus เมื่อปิด ความต่างของสถานะไม่พึ่งสีอย่างเดียว รองรับ zoom 200%, ข้อความยาว และ `prefers-reduced-motion` โดยยังคง feedback ที่จำเป็น

หน้า “ความเป็นส่วนตัว” เขียนสั้นและตรวจสอบได้: “รูปและการแก้ไขอยู่ในอุปกรณ์นี้ เราไม่ขอบัญชี ไม่อัปโหลดรูป ไม่เปิดกล้อง ไม่ใช้ analytics และไม่เก็บข้อมูลส่วนบุคคล การกดบันทึกจะสร้างไฟล์ให้เบราว์เซอร์ดาวน์โหลดลงเครื่อง” ไม่ฝัง CDN fonts, tracking pixels, telemetry หรือ asset requests ที่ส่งรายละเอียดการใช้งานออกไปโดยไม่จำเป็น

# ขอบเขตหลักฐาน

นี่เป็นข้อเสนอการออกแบบสำหรับแอปสมมติ ยังไม่มี implementation, rendered viewport, การทดสอบ interaction/accessibility, การประเมินด้านกฎหมาย หรือการทดสอบกับเด็กจริง จึงยังไม่ควรกล่าวว่าใช้ง่าย ปลอดภัย เข้าถึงได้ เป็นไปตาม COPPA หรือได้รับการยืนยันจากผู้ใช้ การตรวจขั้นต่อไปควรรวมการทดสอบที่ประมาณ 360, 768, 1280 และ 1600 px; touch, pen, mouse และ keyboard; ทุก state ด้านบน; และ usability research ที่ได้รับความยินยอมอย่างเหมาะสมกับเด็กและผู้ปกครอง
