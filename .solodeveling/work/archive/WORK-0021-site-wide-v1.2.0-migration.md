## Goal

Make every public Gridgeist surface honestly reflect v1.2.0 and inherit one shared
system contract without flattening the distinct jobs of the homepage, docs,
examples, case study, social card, or README showcase.

## Direction

A public field manual for builders and reviewers that makes the Gridgeist method
visible through a rational grid, restrained editorial proof, cobalt/acid state
signals, and authentic system-contract evidence.

The direction is **brand-derived** from the converging public implementation,
wordmark, paper/ink/cobalt/acid palette, heavy Inter display typography, DM Mono
operational labels, visible alignment grid, square surfaces, and evidence-first
voice. The user authorized evolving this reviewed recommendation.

## Compact v1.2 system contract

- **Color:** Foundation palette maps to canvas, surface, text, muted, border,
  action, focus, success, warning, and destructive semantic roles. Component rules
  consume semantic roles; status and current states retain non-color cues.
- **Typography:** Inter owns display/heading/body; DM Mono owns commands, versions,
  metadata, and compact labels. Thai overrides retain appropriate line height and
  word breaking without importing a new font dependency.
- **Layout and spacing:** One shared outer gutter and spacing scale; visible grid on
  thesis/evidence surfaces, quiet grid for docs/case narrative, and explicit mobile
  re-composition rather than desktop shrinkage.
- **Shape and surface:** Square adjacent surfaces and explanatory rules remain the
  default. Radius, shadow, and gradient stay confined to intentional generic Before
  examples or functional mobile overlays.
- **Components:** Header/navigation, buttons, version marks, evidence strips,
  command/copy controls, tabs, status labels, section headers, and proof modules use
  shared anatomy and applicable default/hover/focus/active/current/copied states.
- **Motion:** Pointer axes, reveals, and view transitions communicate relationship
  or hierarchy; reduced motion disables nonessential movement without hiding state.

   Tracefield inherit the system while preserving their distinct content and truth.

   overflow and retains a clear dominant hierarchy. English/Thai navigation,
   comparison controls, details, copy controls, focus, reduced motion, and relevant

7. Browser console has no application errors; any third-party or hosting-only gap is
   recorded separately. Release/skill validation, JavaScript syntax, HTML route
   audit, image metadata, and diff integrity pass.

   untracked files remain untouched. No push or deploy occurs without separate user

## Risks and recovery

- Shared CSS can regress dense page-specific layouts; keep page CSS authoritative
  for composition and verify every route after each shared-system slice.
- Thai expansion can expose clipping hidden in English; inspect Thai independently
  at narrow and wide widths.
- Evaluation copy can overstate evidence; preserve the distinction between the
  published scenario suite, six counted v1.2 fixture runs, repaired guardrails, and
  real-user evidence not established.
- Recover by reverting the single migration commit before publication; generated
  OG and all source changes remain version-controlled.
