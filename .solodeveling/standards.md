# Standards

## Definition of Done

- Follow `AGENTS.md` and the active Solodeveling work item.
- Keep installable skill frontmatter and agent metadata valid.
- Keep plugin manifest, marketplace ref, changelog, and Git tag version-aligned for releases.
- Update English and Thai user guidance together.
- Run changed-scope checks plus the broadest practical validation.
- Record observed evidence and explicit gaps before closing tracked work.

## Release conventions

- Use Semantic Versioning.
- Treat compatible capability expansion as a minor release.
- Pin public marketplace plugin sources to the release tag.
- Publish GitHub Release notes with update commands near the top.
- Start a new Codex thread after reinstalling an updated plugin.
